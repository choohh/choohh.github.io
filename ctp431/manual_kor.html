코드 쌓기 프로그램 한글 사용 설명서

20160680 추헌호 (화학과)

 

I. 어떤 프로그램인가요..?

  <코드 쌓기 프로그램>은 기타 코드를 잡는 다양한 폼과, 그 이면에 있는 화성학적 개념을 학습할 수 있도록 도와주는 프로그램입니다. 이 프로그램은 대중이 아닌, 어느 정도 기타를 연주할 줄 아는 아마추어 기타리스트를 위한 것입니다. 즉 기타 지판상의 위치와 음정의 관계, 코드의 화성과 구성음 등에 대한 기본 지식을 가진 사람을 대상으로 만들었습니다. 서로 어울리는 단음들로 이루어진 '코드'의 종류는 매우 많습니다. 더욱이 같은 코드라도 기타 지판상에서 표현될 수 있는 방법은 무궁무진합니다. 그러나 이 많은 기타 코드를 처음부터 다 외울 수는 없습니다. 그래서 기타를 처음 배우는 사람들은 보통 C, F, G, A 등 가장 기초적이고 잡기 쉬운 코드의 폼을 외우면서 코드를 처음 접합니다. 그 다음으로는 5번 또는 6번줄이 근음을 담당하는 형태의 정형화된 코드 폼을 외워서, 같은 코드 폼을 위치만 바꾸면서 사용합니다. 예를 들면 Bm7과 Cm7 코드를 각각 잡을 때 손모양은 같지만 지판상의 손의 위치는 다릅니다.
  이 한 단계 진보한 방법은 몇 개의 폼을 외움으로써 굉장히 많은 코드의 폼을 적어도 한 개는 알게 해 준다는 장점이 있습니다. 그러나 이렇게 정형화된 폼을 외우는 방식은 곡을 이루는 코드의 근음들이 지판상에서 멀리 떨어져 있을 때, 손을 부산스럽게 움직일 수밖에 없도록 함으로써 연주의 정확도를 떨어뜨리는 단점이 있습니다. 즉, 한 곡 안에서의 코드들은 서로 붙어 있어야 기타리스트가 연주하기 편합니다. 그렇기 때문에 연주자들은 비슷한 위치에 여러 코드들이 몰려있는 형태의 폼을 연구하는데, 이렇게 코드의 폼을 바꾸고 때로는 구성음을 유지하면서 근음까지 바꾸는 것을 코드 보이싱이라고 합니다.
  코드 보이싱을 연습하는 가장 좋은 방법 중 하나는 기타 지판에서 직접 다양한 코드 폼을 고안해보는 것입니다. 이 과정에서 아마추어 기타리스트는 단순히 코드의 폼을 익히는데 그치지 않고, 코드의 구성음을 구별하는 화성학 지식과 기타 지판상의 위치에 따른 음정에 대한 감각을 습득할 수 있습니다. 그러나 이 과정을 혼자 시작하기에는 다소 부담스러울 수 있다는 것이 제가 이 프로그램을 만들게 된 계기입니다. 이 프로그램은 무작위로 코드와, 코드 폼이 위치할 프렛의 범위를 지정해줍니다. 그러면 플레이어는 지정된 범위 안에서 지정된 코드의 폼을 만들어야 하고, 채점도 받을 수 있습니다. 이를 통해 많은 아마추어 기타리스트들이 코드 공부를 하는데 도움을 받을 수 있을 것입니다.

 

II. 어떻게 사용하나요..?

  1. 기타 지판 위에서 특정 줄의 특정 프렛을 클릭하면 그 지점에 해당하는 음정의 소리가 납니다. 이곳 저곳을 클릭하면서 직접 해보세요! 개방현을 치고 싶다면 원하는 줄높이에 해당하는 지판 왼쪽 하늘색 여백을 클릭하세요.
  2. Choose Chord 버튼을 누르세요. 게임이 시작되었습니다! 넙죽이가 랜덤하게 코드 하나를 말해줄 것입니다. 동시에 기타 지판 위에서 일정 부분이 랜덤하게 어두워질거에요.
  3. 정해진 범위 안에서는 마우스를 클릭한 위치에 해당하는 음을 들을 수 있을 뿐 아니라 흰색 원으로 표시할 수 있어요! 이미 표시한 원을 지우고 싶다면 한번 더 클릭하세요. 당연히 하나의 줄에서는 하나의 원밖에 표시할 수 없습니다.
  4. 흰색 원으로 표시해서 넙죽이가 말해준 코드의 폼을 완성하세요!
  5. Submit 버튼을 누르면 자동으로 채점됩니다. 만약 넙죽이가 말해준 코드에 맞지 않는 음이 포함되었거나 구성음이 빠져있다면 "Wrong! Try again."이라는 메시지가 뜰 거에요.
  6. 만약 성공적으로 코드 폼을 완성했다면 지판이 다시 깨끗해지고 진행 상황(Progress)은 1, 점수(Score)는 10 올라갑니다. ;-) 또 코드 폼 완성까지 걸린 시간도 보여집니다.
  7. 계속 진행하고 싶으시다면 다시 Choose Chord 버튼을 눌러서 새 코드를 받으세요!

 

III. 알고리즘 특징

  1. 기타 프렛간 간격은 로그 스케일을 따릅니다. 이 프로그램은 화면의 가로, 세로 픽셀수를 인식해서 자동으로 화면에 맞는 지판과 프렛들을 표시해줍니다.
  2. A4 = 440Hz는 1번줄 5번프렛에 해당되며, 이를 기준으로 각 위치의 음정의 진동수가 설정되어 있습니다. 지판을 클릭했을 때 나오는 소리는 Web Audio API를 통해 만들어진 triangle type의 파동입니다.
  3. 코드의 구성음간에는 일정한 규칙이 있습니다. 예를 들어, 마이너(min) 코드는 근음과 단3도, 완전5도로 구성되어 있습니다. 이 프로그램은 코드가 주어지면 근음과 코드 타입을 통해 자동으로 구성음을 계산하고 이를 채점에 활용합니다.
  4. p5.js가 사용되었습니다. p5의 draw()는 1초에 60번 지판 상태를 체크하고 흰색 원을 그려주는 역할을 함으로써 마우스 클릭을 통해 원을 표시했다 지울 수 있도록 해 줍니다.

 

IV. 한계와 개선 방향

  (너무 늦게 생각해서 구현할 시간이 없었거나, 아직 프로그래밍 능력이 부족하여 완성하지 못한 것들..T_T)

  1. 현재 프로그램은 EBGDAE 스탠다드 튜닝 상태의 기타만에서 사용될 수 있다는 한계가 있습니다. 다양한 다른 튜닝 상태를 옵션으로 넣으면 활용도가 더 높아질 것입니다.
  2. 지판을 클릭했을 때 나오는 소리의 파형이 단순한 triangle type이라는 한계가 있습니다. Waveform customizing을 통해 진짜 기타 소리와 비슷한 파형을 만들고, 더 나아가 Fender, Gibson 등 다양한 기타 브랜드의 소리를 각각 구현할 수 있다면 플레이어에게 더 현실감을 줄 수 있을 것입니다.
  3. 프렛 범위는 4개로 설정되어 있는데, 이는 프렛간 거리가 5개 이상으로 늘어나면 손으로 잡기 힘들다는 이유 때문이었습니다. 그러나 이는 사실 너무 단순한 생각입니다. 같은 프렛 범위 안에서도 더 잡기 쉽게 코드 폼을 완성했을 경우, 더 높은 점수를 줄 수 있도록 개선해야 합니다. 이를 위해서는 어떤 구조의 폼일 때 손으로 잡기가 더 편한지 추가 연구가 필요합니다.
  4. 새로고침을 누르면 플레이했던 데이터가 다 날아간다는 한계가 있습니다. 서버를 구축해서 자신의 플레이 기록을 볼 수 있도록 개선할 수 있습니다.
  5. 힌트 버튼을 만들어서 그것을 누르면 코드의 구성음이나, 코드에 따른 화성 정보("마이너는 근음, 단3도, 완전5도"와 같이)를 힌트로 줄 수 있게 개선할 수 있습니다.
